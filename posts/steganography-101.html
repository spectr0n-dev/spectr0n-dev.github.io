Steganography 101: Hiding Data in Plain Sight

2026-02-18 — spectr0n

Steganography is the practice of hiding information within other non-secret data. Unlike encryption, which makes data unreadable, steganography makes data invisible.

The most common technique is hiding data inside image files. JPEG images are particularly good carriers because they're common, often large, and small modifications are imperceptible to the human eye.

Using steghide

My tool of choice is steghide. It's simple, effective, and supports JPEG and BMP formats with AES encryption.

# Embed a secret file inside an image
steghide embed -cf photo.jpg -ef secret.txt -p "passphrase"

# Extract hidden data from an image
steghide extract -sf photo.jpg -p "passphrase"

# Check if an image contains hidden data
steghide info photo.jpg
Pro tip: If you use an empty passphrase (-p ""), steghide will still embed the data — it just won't be encrypted. Useful for quick dead drops where the obscurity of the carrier is the security layer. I sometimes use this for my own personal notes embedded in blog header images.

The beauty of steganography is that nobody thinks to look. A JPEG image of a city skyline could contain an entire document, and no one would ever know unless they specifically checked with the right tool.

Detection

Tools like stegdetect and binwalk can sometimes detect steganographic content, but steghide's algorithm is designed to resist statistical analysis. The embedded data is scattered across the least significant bits of the DCT coefficients.

Practical applications

  • Whistleblower communications
  • Backup passphrases (hide a passphrase inside a family photo)
  • Dead drop information exchanges
  • Plausible deniability (the image looks normal even if inspected)

Remember: the best hiding place is in plain sight.
